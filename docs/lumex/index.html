    Basic Concept

    Controller

    Instead of defining all of your request handling logic as Closures in route files, you may wish to organize this behavior using Controller classes. Controllers can group related request handling logic into a single class. Controllers are stored in the app/Http/Controllers directory.

    Below is an example of a basic controller class. Note that the controller extends the base controller class included with Laravel.

    <?php namespace App\Http\Controllers; use App\User; use App\Http\Controllers\Controller; class TestController extends Controller { /** * Show the profile for the given user. * * @param int $id * @return Response */ public function GETprofile($id) { return view('user.profile', ['user' => User::findOrFail($id)]); } }

    Laravel Info

    For full documentation about how to work with laravel controller, follow this documentation:
    Laravel Controllers

    Model

    The Eloquent ORM included with Laravel provides a beautiful, simple ActiveRecord implementation for working with your database. Each database table has a corresponding "Model" which is used to interact with that table. Models allow you to query for data in your tables, as well as insert new records into the table.

    Models typically live in the app directory, but you are free to place them anywhere that can be auto-loaded according to your composer.json file. All Eloquent models extend IlluminateDatabaseEloquentModel class.


    Lumex auto generate model for module. Model placed in App/Models directory

    Below is the example of model class

    <?php namespace App; use Illuminate\Database\Eloquent\Model; class TestModel extends Model { /** * The table associated with the model. * * @var string */ protected $table = 'my_table'; }

    To get model in controller simply

    use App\Models\TestModel;

    Laravel Info

    For full documentation about how to work with laravel eloquent model, follow this documentation:
    Laravel Eloquent

    View

    Views contain the HTML served by your application and separate your controller / application logic from your presentation logic. Views are stored in the resources/views directory. A simple view might look something like this:

    <!-- View stored in resources/views/hello.blade.php --> <html> <body> <h1>Hello, {{ $name }}</h1> </body> </html>

    Below is the example to call view in the controller:

    $this->data['name'] = "Allan"; return view('hello', $this->data);

    Lumex auto generate view into resources/views/module_name/ directory.

    Working with Ajax

    Request

    This framework sending all request via ajax for any method like POST, GET, DELETE.

    To send request simply follow this rule

    main#module/function/param-1/param-2/param-n?query-string

    which is:

    main#
    mandatory, main container for ajax content, containing sidebar menu, top menu, application info etc.
    module
    mandatory, the module name to be called by ajax
    function
    the class function/method to be executed
    param-1, ... param-n
    optional, the aditional param for executing command
    query-string
    optional, the query string like query1=value1& etc.

    We can call this request via hyperlink, form action or php redirect

    Request Via Request Methods Example
    Hyperlink GET <a href="main#my_module/function">some link</a>
    Form GET, POST, DELETE, PUT <form action="main#my_module/function">
    <input type="hidden" name="_method"="PUT">
    ...
    <button type="submit">Send</button>
    </form>
    Redirect GET return redirect('/main#my_module/function');

    Routing

    All Laravel routes are defined in your route files, which are located in the routes directory. These files are automatically loaded by the framework. The routes/web.php file defines routes that are for your web interface.

    Lumex framework generating modules route located in routes/module directory with the module name as route filename.

    Each module will have minimum set of routes like below

    Route::delete('module/{id}','moduleController@destroy')->where('id','.*'); Route::put('module/','moduleController@update'); Route::get('module/{any}', 'moduleController@implicitRoute')->where('any','.*'); Route::post('module/{any}', 'moduleController@implicitRoute')->where('any','.*'); Route::resource('module','moduleController', ['only'=>['index','store','update','delete']]);

    Feel free to add additional route for your future need.

    Validation

    Laravel provides several different approaches to validate your application's incoming data. By default, Laravel's base controller class uses a ValidatesRequests trait which provides a convenient method to validate incoming HTTP request with a variety of powerful validation rules.

    To do the validation, we will use the validate method provided by the IlluminateHttpRequest object. If the validation rules pass, your code will keep executing normally; however, if validation fails, an exception will be thrown and the proper error response will automatically be sent back to the user.

    In the case of a traditional HTTP request, a redirect response will be generated, while a JSON response will be sent for AJAX requests.

    Below is example to do validatino in laravel

    $request->validate([ 'title' => 'required|unique:posts|max:255', 'body' => 'required', ]);

    Hashing ID

    Hashids is a small open-source library that generates short, unique, non-sequential ids from numbers.

    It converts numbers like 347 into strings like “yr8”, or array of numbers like [27, 986] into “3kTMd”.

    You can also decode those ids back. This is useful in bundling several parameters into one or simply using them as short UIDs.

    Lumex using HashID to generate item ID

    For example instead of using product/show/347, lumex use product/show/yr8 in url path.

    Add relationship from relationship wizzard and lumex will automate the code production.

    Information to be entered are:

    • Model, model to be joined to master model, required
    • Relationship Name, name of relationship, usually model name, required
    • Relationship Type, one-to-one or one-to-many, required
    • Foreign Key, primary key of foreign model/table, required
    • Local Key, field name connected to foreign key, required

      Example:

      webHelper::force_redirect('/login');

      Return value: Object

    • Get_Dir ( $path , $include_file )

      get directory content of specific path

      Example:

      webHelper::hashID( 347 );

    • Decode_ID( $encoded_ID )

      Decode ID with HashID algorithm

      Example:

      webHelper::hashID( "3kTMd" );

      Return value: array [ 27 , 986 ]

      Render twig template to html

    Data Transform

    Transform manually (outside the generator) output from database to another patterned output

    Controller

    To make easy to track, transform formula placed in controller.

    There are 2 method to perform this tranform

    transform_model_rows to transform multiple rows.

    transform_model_attribute to transform single row.

    Example

    <?php namespace App\Http\Controllers; use App\Http\Controllers\Controller; use Illuminate\Http\Request; use Illuminate\Support\Facades\Response; class testController extends Controller { // previous code // ... /* * here you can transform attribute for show */ public function transform_model_attribute( & $model ) { // transform attribute formula here // ... } /* * here you can transform each row in data lists */ public function transform_model_rows(& $model ) { foreach ($model as $key => & $row) { // transform attribute formula here // ... }; } }

    Multiple Rows

    Tranforming multiple rows can be done with the example code below:

    Example

    public function transform_model_rows(& $model ) { // change every rows that contain specific data item foreach ($model as $key => & $row) { // change image url data to img tag $row->image = '<img src="' . asset('assets/image/' . $row->image) . '" >'; // change url link data to hyperlink $row->link = '<a href="www.mysite.com/' . $row->title .'" >Link</a>'; }; }

    Single Rows

    Tranforming multiple rows can be done with the example code below:

    Example

    public function transform_model_attribute( & $model ) { // change date format $model->booking_date = date('j F YYY', $model->booking_date); // change number format $model->payment = format_number($model->payment, ',','.'); }

    Storage

    Introduction

    Laravel provides a powerful filesystem abstraction thanks to the wonderful Flysystem PHP package by Frank de Jonge. The Laravel Flysystem integration provides simple to use drivers for working with local filesystems, Amazon S3, and Rackspace Cloud Storage. Even better, it's amazingly simple to switch between these storage options as the API remains the same for each system.

    To create the symbolic link, you may use the storage:link Artisan command:

    php artisan storage:link

    Of course, lumex has already doing for you, so you don't need to do it anymore.

    Laravel Info

    For full documentation about how to work with laravel filesystem, follow this documentation:
    Laravel File System

    Upload File

    In web applications, one of the most common use-cases for storing files is storing user uploaded files such as profile pictures, photos, and documents. Laravel makes it very easy to store uploaded files using the store method on an uploaded file instance. Simply call the store method with the path at which you wish to store the uploaded file:

    <?php namespace App\Http\Controllers; use Illuminate\Http\Request; use App\Http\Controllers\Controller; class UserAvatarController extends Controller { /** * Update the avatar for the user. * * @param Request $request * @return Response */ public function update(Request $request) { $path = $request->file('avatar')->store('avatars'); return $path; } }

    If you would not like a file name to be automatically assigned to your stored file, you may use the storeAs method, which receives the path, the file name, and the (optional) disk as its arguments:

    $path = $request->file('avatar')->storeAs( 'avatars', $new_file_name );

    For local storage, uploaded file store in storage/app directory

    Retrieving File

    The get method may be used to retrieve the contents of a file. The raw string contents of the file will be returned by the method. Remember, all file paths should be specified relative to the "root" location configured for the disk:

    $contents = Storage::get('file.txt');

    For local storage, uploaded file will be retrieved from storage/app folder

    Extending Controller

    Controller

    In laravel framework, every time we create controller, it always extends to App\Http\Controllers\Controller.

    The content of controller as below:

    <?php namespace App\Http\Controllers; use Illuminate\Foundation\Bus\DispatchesJobs; use Illuminate\Routing\Controller as BaseController; use Illuminate\Foundation\Validation\ValidatesRequests; use Illuminate\Foundation\Auth\Access\AuthorizesRequests; class Controller extends BaseController { use AuthorizesRequests, DispatchesJobs, ValidatesRequests; }

    How if we need to extends more than single file BaseController?

    We do it by using trait class

    Trait

    Trait like a class, or it still a class.

    The different is we can not extend trait with standard class extending

    class myClass extends otherClass

    To call trait we using use statements

    namespace App\Http\Controllers; use Illuminate\Routing\Controller as BaseController; use App\Traits\myTrait; use App\Traits\myOtherTrait; class Controller extends BaseController { use myTrait, myOtherTrait; }

    PHP Info

    For complete information about trait, refer to

    PHP Trait

    Make Your Own (for advanced developer)

    To make your own trait, use following example

    <?php namespace App\Traits; Trait MyTrait { public function hello_world(){ return 'hello world'; } }

    save filename as app/Traits/MyTrait.php


    include in base controller

    <?php namespace App\Http\Controllers; use Illuminate\Foundation\Bus\DispatchesJobs; use Illuminate\Routing\Controller as BaseController; use Illuminate\Foundation\Validation\ValidatesRequests; use Illuminate\Foundation\Auth\Access\AuthorizesRequests; use App\Traits\MyTrait; class Controller extends BaseController { use AuthorizesRequests, DispatchesJobs, ValidatesRequests, MyTrait; }
